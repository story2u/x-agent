# 开发计划

## 项目定位

x-agent 是一个本地优先的 AI Agent Client。

当前主形态是 CLI/TUI，不是 Web 后台：

- Chat（单输入框）是主入口。
- Slash commands 是导航方式。
- Skills 使用本地 `skills/*/SKILL.md`。
- References 使用 skill 目录下的 Markdown 文件。
- Agent Runtime 负责加载 skill、references、tools、run context，并产出 artifact。
- Web 登录、D1、Durable Object、Cloudflare Workers 后台审批流已退出当前 MVP 主路径。

长期目标：

> 用本地 TUI + SKILL.md + references + tools + eval harness，构建可复用、可审计、可演化的 AI Agent 工作流。

当前重点垂直方向：

> `daily-fortune-tweet`：以赛斯资料作为意识解释内核，结合东西方命理象征系统，生成适合 X/Twitter 发布的今日运势长推文和 thread。

## 当前 MVP 状态

已完成（基础设施）：

- 主操作入口为本地 CLI/TUI：`npm run tui`。
- 单输入框 client：普通输入生成文本，其他操作通过 `/` slash commands（`/skills` `/skill` `/tone` `/output` `/audience` `/goal` `/constraints` `/model` `/last` `/history`）。
- 模型默认 `openai-codex`（ChatGPT Plus/Pro OAuth），支持 `PI_PROVIDER=deepseek`、`openai`。
- Skills 切换为本地 `skills/*/SKILL.md`，不再以 D1 为 source of truth。
- 内置 `twitter-launch-creative` 和 `daily-fortune-tweet`。
- 本地 skill 支持 `references/*.md` 注入 runtime prompt。

已完成（daily-fortune-tweet v3，星座垂直切片）：

- **5 段独立推理 pipeline**：understand → diverge → judge → draft → refine（`src/lib/fortune/pipeline.ts`），取代旧的单次成稿 + 自评表演。judge 段独立评审，不再由写稿者自评。
- **确定性星象引擎** `src/lib/fortune/astro-day.ts`：weekday/weekdayPlanet、moonPhase、sunSeason、signProfile、focusDomain、emotionalWeather、dailySeed。
- **共享模型层** `src/lib/pi-model.ts`（resolveModel/streamForModel/getModelApiKey），单次路径与 pipeline 复用。
- **命理底料 references**：`astrology-signs.md`、`astrology-daily-engine.md`，以及既有运营/hook/audience/safety/golden-examples。
- **长度兜底**：longTweet/both 若正文 < 600 中文字，确定性触发 `expand` 段扩写到 600-1200 字。
- **运营级内容结构**：Audience Insight、Angle Mining、Hook Lab、Fortune Spine、Operator Critique、Rewrite、Engagement Plan、Review Notes、Long Tweet / Thread Output Contract。
- **两层 eval**：
  - `npm run eval:skills`（`scripts/eval-skills.ts`）—— 只校验 eval spec 形状。
  - `npm run eval:fortune`（`scripts/eval-fortune-run.ts`）—— **真跑** pipeline + 确定性规则 + LLM-judge + 运营达标率（需凭据）。
- **提速开关** `FORTUNE_THINKING_CAP=low|medium|high`：统一封顶各段 thinking，用于 eval/迭代提速。

不在当前 MVP：图片生成、图片方向、alt text、media upload、自动发推、Web 后台。

## 当前核心问题

`daily-fortune-tweet` 的运营 pipeline 已成型、真 eval 已可量化达标率，但要成为稳定的差异化产品，底层系统还需升级：

### 1. 命理底料仍偏轻，且层次未分

当前更准确的定位是「确定性星象创意 seed + 运营内容 pipeline」，**还不是**「完整专业命理推算系统」。需要继续沉淀东西方象征底料，并明确分层。

### 2. 赛斯资料还没有成为意识内核

差异化方向不是传统黄历，也不是普通星座号，而是：

> 星象、节气、五行、塔罗、生肖作为**象征语言**；赛斯资料作为**意识解释内核**；内容最终回到注意力、信念、概率线、选择点、当下行动。

运势内容不应说「未来一定发生什么」，而应帮助用户看见：今天注意力适合放哪、哪个信念在影响体验、当前处在哪个选择点、情绪给了什么信号、今天做什么小行动能让下一条概率线更稳。

### 3. 术语未治理：创意种子被误当命理事实

`focusDomain`、`emotionalWeather` 是 `dailySeed` 哈希轮换出来的**创意种子**，却与真天文量（`moonPhase`/`sunSeason`）平级摆放。需要标注每个变量的来源与置信度，避免"伪高精度命理"。

### 4. date/timezone 未显式进入 pipeline

`getAstroDay` 当前直接吃 `new Date()`，不可复现、时区盲，pipeline 测试不可控。

### 5. eval 待补 mock

`eval:fortune` 已能真跑评测，但依赖真实模型调用；需补 dry-run/mock 以便无凭据/CI 场景，并拆分 spec/live 命名。

## 命理底料五层架构

```text
Layer 1: Deterministic Time Layer    日期/时区/星期/月相/太阳季/节气/农历/生肖年
Layer 2: Symbolic Divination Layer   星座/月相/塔罗/五行/生肖/节气/黄历象征
Layer 3: Seth Consciousness Layer    信念/注意力/概率线/当下力量点/选择点/小行动
Layer 4: Content Operation Layer     audience/angle/hook/longtweet/thread/critique
Layer 5: Safety Layer                娱乐化/非确定性/不制造恐惧/不给投资医疗法律赌博建议
```

原则：

1. 时间与命理变量提供**象征底料**，不用于确定性预测。
2. 赛斯资料是**意识解释层**，不是预测引擎。
3. 星象/五行/节气/塔罗是**象征语言**，提供隐喻和内容角度。
4. 输出必须把主动权还给用户，避免「命中注定」「不可改变」。
5. 对外口径：**确定性象征底料 + 赛斯意识解释 + 运营内容 pipeline**；不宣称具备完整专业命理推算能力。

### Layer 1 Deterministic Time Layer
可复现、可测试的时间变量；近似算法必须标 approximate；creative rotation 不得伪装成 astrology fact；date/timezone 显式进入 pipeline。

### Layer 2 Symbolic Divination Layer
西方：星座画像、太阳季、星期主行星、月相、塔罗 one-card seed（后续行星/宫位/相位/逆行）。东方：农历、生肖、节气、五行、干支、黄历、易经（先 reference 后计算）。所有传统符号翻译成现代行动建议，不直接输出传统「宜忌」。

### Layer 3 Seth Consciousness Layer
赛斯资料**只作内部写作透镜**：信念塑造感知、注意力组织经验、当下是力量点、未来是概率性的、情绪是信号、外部象征是内在的镜子、小行动可改变下一条概率线。去宿命、去恐惧、给一个小行动、把主动权还给读者。新增 `references/seth-consciousness-framework.md`。

> 版权/口径护栏：赛斯资料是 Jane Roberts 的特定版权文本。只用**概念**作改写式写作指南，不大段逐字引用、不归因具体话语（「赛斯说今天…」）、对外不宣称权威传达赛斯。

### Layer 4 Content Operation Layer
已有运营结构。下一步：每段记录结构化 trace；judge 独立评审；refine 明确检查 Seth agency framing；longTweet/thread 分别策略；golden examples 扩到 10-20 条。

### Layer 5 Safety Layer
禁止确定性预测、保证发财/脱单/升职、医疗/法律/投资/赌博建议、恐惧操控、「不转会倒霉」、把 creative seed 包装成命理事实。允许娱乐运势、轻玄学氛围、情绪陪伴、自我反思、注意力提醒、行动建议、选择点提示、概率线隐喻。

## 近期开发计划（切片执行，不一次做完）

> 切片顺序经评审确定：先做高价值、低风险、可测、少烧额度的 Slice 1，再逐步推进。

### Slice 1（P0 + P1 + P2）— 术语治理 + date/timezone + 赛斯 reference

- **P0 术语治理**：新增 `FortuneSourceLevel` / `FortuneConfidence` / `FortuneFactor`；给 weekdayPlanet（traditional-symbolic/high）、moonPhase（approximate-astronomical/medium）、sunSeason（deterministic-calendar/high）、signProfile（traditional-symbolic/medium）等标注来源；把 `focusDomain` → `creativeFocusDomain`、`emotionalWeather` → `creativeEmotionalWeather`（creative-rotation/creative），**仅改 astro-day 输入种子，不动 `DailyFortuneArtifact.fortuneSpine.emotionalWeather` 等输出契约**；保留兼容 alias。
- **P1 date/timezone**：`getAstroDay`/pipeline 接受显式 `date`(YYYY-MM-DD) 与 `timeZone`(IANA)；命令可解析日期；缺省用配置/系统；测试固定 date/timezone 复现。
- **P2 赛斯 reference**：新增 `references/seth-consciousness-framework.md`；SKILL.md 与 pipeline 的 diverge/draft/refine 段读取；输出体现注意力/信念/概率线/当下力量点/选择点/小行动其一以上；refine/safety 检查 fatalism。

验收：typecheck/lint/test 通过；输出出现 agency framing、无 fatalistic 表达；sourceLevel/confidence 有测试覆盖；同 date+sign 输出稳定、不同 date 变化。

### Slice 2（P4 + P5）— 统一 FortuneContext + stage trace
统一 western/eastern/seth/creative 四层为 `FortuneContext`，`resolveFortuneContext(input)` + `formatFortuneContextForPrompt`；注入各 stage；新增结构化 `FortunePipelineTrace`（stage/summary/inputKeys/outputKeys/selectedReferences/scores/warnings），`/last` 或 debug 可看；不暴露完整私有推理原文。

### Slice 3（P3）— 东方象征时间层 MVP
新增 `src/lib/fortune/eastern-day.ts` + `references/eastern-symbolic-calendar.md`：生肖年（春节边界小表，否则标 approximate）、节气（静态表，deterministic-calendar + ±1 天备注）、五行象征→现代行动、季节建议。干支日/八字/紫微暂不计算。

### Slice 4（P6）— Live eval 收尾（基于现有 `eval:fortune` 扩展，不重建）
保留 `eval:fortune` 真跑能力；补 **dry-run/mock** 模式（无凭据/CI 可用）；按需拆分命名（如 `eval:skills:spec` / `eval:skills:live`）。live 检查项：`final.longTweet.body` 存在、600-1200 中文字、含 FortuneContext 变量、含 Seth agency framing、≥2 具体场景、含 angleOptions/hookOptions/operatorCritique/engagementPlan、无禁止词、thread 5-8 条、operator scores 达标、safety 通过。

### 暂缓
完整东方计算、golden examples 扩到 10-20、通用化 pipeline skill、本地知识库（SQLite/FTS/检索）。

## 不做事项

- 不回到 Web 后台主形态，不重建 Skill Studio Web 编辑器。
- 不做确定性预测，不保证财运/感情/事业/健康结果。
- 不提供投资/医疗/法律/赌博建议，不制造恐惧或迷信压力。
- 不宣传完整专业命理推算能力，不伪装高精度星盘/八字计算。
- 不把 creative seed 当成真实命理事实，不直接复制传统黄历。
- 不把赛斯资料当预测模型，不大段逐字引用赛斯文本。
- 不做图片生成/media upload/自动发推，不为功能堆砌引入重型依赖。
