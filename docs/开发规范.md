# 开发规范

## 产品边界

MVP 聚焦 X/Twitter 文本创意工作流：

- 输入自然语言需求和上下文。
- 通过 pi agent 生成推文、hashtags、rationale、review notes。
- TUI client 在当前会话内展示结果。
- 通过 `/` slash commands 管理 skill、tone、output、audience、goal、constraints。

当前不做图片生成、图片方向、alt text、media upload。后续要恢复图片能力，必须先更新 `docs/features/text-agent.md`、`docs/架构文档.md` 和 `docs/数据流程.md`。

唯一操作入口是本地 CLI/TUI（`npm run tui`）。没有 Web 服务、数据库或部署后端。

## 编码边界

- 优先使用现有模块和模式，不新增不必要抽象。
- CLI/TUI operator 交互集中在 `scripts/x-agent-tui.ts`。
- 本地 skill 放在 `skills/<slug>/SKILL.md`，通过 `src/lib/skills/local-skills.ts` 加载。
- agent / 业务逻辑放入 `src/lib/**`。
- agent prompt / tool schema 变更必须同步更新 `src/lib/pi-agent.ts`、测试和 `docs/features/text-agent.md`。
- 模型凭据逻辑变更必须同步更新 `src/lib/pi-credentials.ts`、必要的 `src/lib/pi-agent.ts` provider 分支和 `docs/features/model-oauth.md`。
- 不允许把 skill source of truth 迁回任何数据库。

## 本地开发

常用命令：

```bash
npm run tui
npm run build
npm run typecheck
npm run lint
npm test
```

凭据放在本地 `.env`（已 gitignore），不提交。

## 质量门

按风险选择验证：

- 纯文档：不要求测试，但要检查文档链接和功能导航一致。
- CLI/TUI 改动：至少 `npm run tui -- --help`、`npm run lint`、`npm run typecheck`。
- 本地 skill / agent runtime 改动：至少 `npm run typecheck`、`npm run lint`、`npm test`。
- 提交前：确认 git 工作区范围，避免提交无关生成文件或 `.env`。
- 涉及模型 provider 或 Daily Fortune runtime：至少 `npm run build`、`npm run typecheck`、`npm run lint`、`npm test`。

## 文档回写

每次实现或修复后必须回写 docs：

- 更新对应 `docs/features/*.md` 的“当前状态”“相关模块”“测试点”。
- 架构或数据流变化同步更新 `架构文档.md` 或 `数据流程.md`。
- 用户流程变化同步更新 `业务流程.md`。
- 新模块或功能入口同步更新 `功能导航.md`。
