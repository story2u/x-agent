# 业务流程

## 文本生成流程

1. Operator 运行 `npm run tui`。
2. TUI 显示一个输入框。
3. Operator 输入自然语言请求，或用 `/` slash commands 调整上下文。
4. TUI 读取本地 `skills/*/SKILL.md`。
5. `src/lib/skills/local-skills.ts` 根据 `/skill` 设置或关键词选择 skill。
6. `src/lib/pi-agent.ts` 编译 prompt 并调用 pi agent loop。
7. 模型通过 `finalize_twitter_creative` 工具返回结构化文本 artifact。
8. TUI 打印 tweet、hashtags、rationale、notes、skill trace。

模型调用失败时（凭据无效、OAuth 刷新失败等），TUI 直接显示真实错误原因，不再返回保底模板。当前流程不登录、不落地存储。

## Slash Command 流程

- `/skills`：列出本地 Markdown skills。
- `/skill auto`：恢复自动 skill 选择。
- `/skill <slug>`：固定使用某个本地 skill。
- `/tone <value>`：调整语气。
- `/output <tweet|thread|longTweet|both>`：调整输出类型。
- `/audience <text>`、`/goal <text>`、`/constraints <text>`：调整生成上下文。
- `/model`：查看模型 provider 和凭据是否存在。
- `/history`、`/last`：查看当前 TUI 进程内结果。

## Daily Fortune 流程

1. 用户输入包含“今日运势 / 运势 / 星座 / 生肖 / fortune / horoscope / zodiac / daily luck”等触发词，或手动 `/skill daily-fortune-tweet`。
2. runtime 加载 `skills/daily-fortune-tweet/SKILL.md`。
3. prompt 强制娱乐/反思定位，不做确定性预测。
4. TUI 展示 Daily Fortune 的 long post、thread、fortune spine、review notes。

## Skill 流程

1. 新增 skill 时创建 `skills/<slug>/SKILL.md`。
2. `SKILL.md` 必须包含 frontmatter、Workflow、Output Contract、Review Checklist / Safety Rules。
3. TUI 启动后通过 `/skills` 查看。
4. 生成时通过 `/skill <slug>` 手动选择，或由 runtime 自动选择。

## Web 边界

Web 登录、Review Gate、后台审批流等旧 Web 后端已移除；x-agent 现在只有本地 TUI。
