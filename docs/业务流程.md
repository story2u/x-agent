# 业务流程

## 文本生成流程

1. Operator 运行 `npm run tui`。
2. TUI 显示一个输入框，并加载 `.x-agent-tui-history` 中的历史输入用于方向键检索。
3. Operator 输入自然语言请求，或用 `/` slash commands 调整上下文。
4. TUI 读取本地 `skills/*/SKILL.md`。
5. `src/lib/skills/local-skills.ts` 根据 `/skill` 设置或关键词选择 skill。
6. `src/lib/pi-agent.ts` 编译 prompt 并调用 pi agent loop。
7. 模型通过 `finalize_twitter_creative` 工具返回结构化文本 artifact（`daily-fortune-tweet` 例外：改走 5 段推理 pipeline，见下方 Daily Fortune 流程）。
8. TUI 打印 artifact、hashtags、rationale、notes、skill trace；Daily Fortune 会优先打印 `final.longTweet` / `final.thread`，顶层 `tweet` 仅作为摘要。
9. 退出 TUI 时保存本次输入历史；生成结果仍只保存在当前进程内。

模型调用失败时（凭据无效、OAuth 刷新失败等），TUI 直接显示真实错误原因，不再返回保底模板。当前流程不登录、不落地存储。

## Slash Command 流程

- `/skills`：列出本地 Markdown skills。
- `/skill auto`：恢复自动 skill 选择。
- `/skill <slug>`：固定使用某个本地 skill。
- `/tone <value>`：调整语气。
- `/output <tweet|thread|longTweet|both>`：调整输出类型。
- `/audience <text>`、`/goal <text>`、`/constraints <text>`：调整生成上下文。
- `/model`：查看模型 provider 和 ChatGPT/OpenAI/DeepSeek 凭据是否存在。
- `/model <openai-codex|openai|deepseek> [model]`：切换当前 TUI 进程使用的模型 provider / model，不回写 `.env`。
- `/history`、`/last`：查看当前 TUI 进程内结果。

## Daily Fortune 流程

1. 用户输入包含“今日运势 / 运势 / 星座 / 生肖 / fortune / horoscope / zodiac / daily luck”等触发词，或手动 `/skill daily-fortune-tweet`。
2. `src/lib/pi-agent.ts` 路由到 `src/lib/fortune/pipeline.ts`（不走单次 `finalize_twitter_creative`）。
3. `src/lib/fortune/astro-day.ts` 为「今天 + 解析出的星座」计算确定性当日星象事实（月相、星期主行星、太阳季、星座画像、今日侧重域、情绪基调），注入每一段推理。
4. 5 段独立模型推理依次执行：understand（brief）→ diverge（角度 + hook）→ judge（独立运营主编真打分挑选）→ draft（fortune spine + 初稿）→ refine（真改写 + 安全 reframe）。
5. 全流程强制娱乐 / 反思定位，不做确定性预测；refine 段做安全改写。
6. 代码侧确定性组装 `DailyFortuneArtifact`；TUI 展示 long post、thread、fortune spine、operator critique、engagement plan 和 review notes。
7. 开发验证：`npm run eval:skills` 校验 eval spec 形状；`npm run eval:fortune` 真跑 pipeline + 规则 + LLM-judge 评真实输出、打印运营达标率（需凭据）。

## Skill 流程

1. 新增 skill 时创建 `skills/<slug>/SKILL.md`。
2. `SKILL.md` 必须包含 frontmatter、Workflow、Output Contract、Review Checklist / Safety Rules。
3. 可在 `skills/<slug>/references/*.md` 放置 runtime references；当前 loader 会按文件名排序并全部注入 prompt。
4. 可在 `skills/<slug>/evals/*.json` 放置机器可检查的质量门规则。
5. TUI 启动后通过 `/skills` 查看。
6. 生成时通过 `/skill <slug>` 手动选择，或由 runtime 自动选择。

## Web 边界

Web 登录、Review Gate、后台审批流等旧 Web 后端已移除；x-agent 现在只有本地 TUI。
